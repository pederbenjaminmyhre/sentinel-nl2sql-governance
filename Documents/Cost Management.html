Cost Optimization Measures Implemented

1. Serverless SQL with Auto-Pause

The database uses the General Purpose Serverless tier with a 60-minute auto-pause delay and a minimum of 0.5 vCores. When no queries are running, the database pauses entirely and incurs zero compute cost. For a portfolio project with sporadic traffic, this reduces SQL costs from ~$100/month (always-on) to $5–15/month. The first query after a pause takes 30–60 seconds (cold start), which is acceptable for a demo.

2. GPT-4o-mini Instead of GPT-4o

The LLM deployment uses GPT-4o-mini (15x cheaper per token than GPT-4o) for both SQL generation and semantic verification. For the relatively simple queries this project handles (single-table SELECTs, basic JOINs), the smaller model performs comparably. At portfolio traffic levels, this keeps LLM costs under $3/month even with heavy testing.

3. Self-Correction Loop Caps

The self-correction loop is capped at 2 retries. Without this cap, a badly-behaving LLM could generate infinite correction cycles, each costing tokens. With the cap, the maximum cost per query is 3 LLM calls (1 initial + 2 corrections + 1 semantic verification), putting a hard ceiling on per-request spend.

4. Gate Pipeline Short-Circuiting

If Gate 1 (lexical) blocks a query, Gates 2, 3, and 4 never execute. This means blocked queries cost zero LLM tokens (G3 skipped) and zero database compute (G4 skipped). For attack attempts and malformed queries, the cost per request is nearly zero.

5. Free-Tier Services Where Possible

App Configuration (free tier), Application Insights (5 GB/month free), and Log Analytics (5 GB/month free) all operate within free allocations. These services provide full functionality at zero cost for portfolio-level usage.

Best Practices for Ongoing Cost Control

Set Azure Budget Alerts

Create a budget in the Azure portal for the sentinel-rg resource group with alerts at 50%, 80%, and 100% of your monthly target. Azure will email you before costs exceed expectations.

Monitor Token Usage

Track Azure OpenAI token consumption in the Azure portal under the Cognitive Services resource. If costs spike, consider:

  • Reducing MaxCorrectionRetries from 2 to 1
  • Caching common queries at the application level
  • Switching G3 (semantic verification) to a cheaper model or disabling it for trusted prompts

Use Demo Mode for Testing

Set Sentinel:DemoMode=true when testing locally or demonstrating the project. Demo mode uses in-memory mocks for both the LLM and database, incurring zero Azure costs while still exercising the full gate pipeline with real TSqlParser validation.

Auto-Pause Tuning

The SQL auto-pause delay is set to 60 minutes. If the database is only used for occasional demos, consider reducing this to the minimum (60 minutes is already the minimum). For sustained testing sessions, the database stays awake and costs accumulate at ~$0.50/hour for compute.

Scale Down When Not Demoing

If the project is not being actively demonstrated, you can:

  • Stop the Function App — eliminates the B1 compute cost ($13/mo) entirely
  • Switch to demo mode — eliminates LLM and database costs while keeping the app responsive
  • Delete the resource group — nuclear option; re-deploy in minutes with Bicep when needed

Production Cost Projection

For an enterprise deployment handling 10,000 queries/month, estimated costs would be: Azure Functions Consumption (~$5), Azure SQL Serverless (~$50–80), Azure OpenAI GPT-4o-mini (~$15–25), APIM (~$0.04). Total: ~$70–110/month — a fraction of the cost of a single analyst's monthly salary.
