A Hard Lesson: Why We Chose Basic over Serverless

Serverless SQL Looked Cheap on Paper — It Wasn't

The initial deployment used Azure SQL Serverless (GP_S_Gen5_1) with a 60-minute auto-pause delay. The theory was sound: the database sleeps when idle and you only pay for active compute. In practice, the Serverless tier charges $0.51/hr for vCore compute plus $0.28/hr for the SQL Server license whenever the database is awake — a combined rate of $0.79/hr.

During a single day of deployment, testing, and debugging, the database accumulated ~12 hours of active time because every API test, health check, and connection attempt reset the 60-minute auto-pause countdown. The result: $9.74 in one day — projecting to nearly $290/month if testing continued at that pace.

The Fix: Basic Tier at $4.99/month Fixed

We switched to the Basic tier (5 DTU), which charges a flat $4.99/month regardless of how long the database is active. There are no per-hour compute fees, no license surcharges, and no surprises. For a portfolio project that serves occasional demos and testing, Basic provides more than enough performance (5 DTUs, 2 GB storage) at a predictable, minimal cost.

Monthly savings: $285/month compared to Serverless running 12+ hours/day.

Key insight: Serverless auto-pause is ideal for production workloads with long idle periods (hours between queries). For development and portfolio projects with frequent short bursts of activity, fixed-price tiers are almost always cheaper because every connection resets the pause timer.

Cost Optimization Measures

1. GPT-4o-mini Instead of GPT-4o

The LLM deployment uses GPT-4o-mini (15x cheaper per token than GPT-4o) for both SQL generation and semantic verification. For the relatively simple queries this project handles (single-table SELECTs, basic JOINs), the smaller model performs comparably. At portfolio traffic levels, this keeps LLM costs under $3/month even with heavy testing.

2. Self-Correction Loop Caps

The self-correction loop is capped at 2 retries. Without this cap, a badly-behaving LLM could generate infinite correction cycles, each costing tokens. With the cap, the maximum cost per query is 3 LLM calls (1 initial + 2 corrections + 1 semantic verification), putting a hard ceiling on per-request spend.

3. Gate Pipeline Short-Circuiting

If Gate 1 (lexical) blocks a query, Gates 2, 3, and 4 never execute. This means blocked queries cost zero LLM tokens (G3 skipped) and zero database compute (G4 skipped). For attack attempts and malformed queries, the cost per request is nearly zero.

4. Free-Tier Services Where Possible

App Configuration (free tier), Application Insights (5 GB/month free), and Log Analytics (5 GB/month free) all operate within free allocations. These services provide full functionality at zero cost for portfolio-level usage.

5. Demo Mode Eliminates External Costs

Setting Sentinel:DemoMode=true swaps the LLM and database to in-memory mocks. The full gate pipeline still runs with real TSqlParser validation, but zero Azure OpenAI tokens are consumed and zero database queries execute. This is ideal for local development, CI testing, and recruiter demos.

Best Practices for Ongoing Cost Control

Set Azure Budget Alerts

Create a budget in the Azure portal for the sentinel-rg resource group with alerts at 50%, 80%, and 100% of your monthly target. Azure will email you before costs exceed expectations.

Monitor Token Usage

Track Azure OpenAI token consumption in the Azure portal under the Cognitive Services resource. If costs spike, consider:

  • Reducing MaxCorrectionRetries from 2 to 1
  • Caching common queries at the application level
  • Switching G3 (semantic verification) to a cheaper model or disabling it for trusted prompts

Scale-to-Zero Architecture

When nobody is using the application, the cost approaches zero automatically:

  • Azure Functions (Consumption) — scales to zero instances; you pay nothing when no requests are in flight
  • Azure OpenAI — per-token billing; zero tokens consumed means zero cost
  • API Management (Consumption) — per-call billing; zero calls means zero cost
  • Key Vault, App Config, Insights — free tier or per-operation; effectively zero when idle
  • Azure SQL (Basic) — the only fixed cost at $4.99/mo (always-on by design)

To eliminate all costs entirely, delete the resource group and re-deploy in minutes with Bicep when needed.
